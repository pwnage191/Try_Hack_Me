### Accessing Through Firefox
- Firefox did not allow me to access the HTTP service running on port 22 because some ports are blocked by default.
- To access the website, I modified the Firefox network settings.
- Open a new tab and type: `about:config`
- Search for: `network.security.ports.banned.override`
- Add `22` as a string value and save it.
- Port 22 is blocked by default in Firefox. I added 22 to the allowed ports override list so I could access the web application running on that port.
- After completing the challenge, remember to remove this setting.

### Privilege Escalation
- The challenge description mentioned a root flag, so the next step was privilege escalation.
- I searched for files with the SUID permission bit set.
- From previous experience with OverTheWire Bandit, I knew that SUID binaries execute with the permissions of the file owner.
- During enumeration, I found that: `/usr/bin/strings` had the SUID bit set.
- I checked for interesting files inside the root directory.
- First, I tried: `/usr/bin/strings /root/flag.txt`
- The file did not exist.
- Then I tried: `/usr/bin/strings /root/root.txt`
- <img width="700" alt="image" src="https://github.com/user-attachments/assets/2d796577-381c-4478-b70a-9b5d3bd2e452" />
- This successfully revealed the root flag.

### Lessons Learned
- Do not assume services are running on their default ports.
- Always inspect page source code for hidden clues.
- Follow every hint carefully; challenge creators often provide the next step indirectly.
- Steganography can be combined with cryptography to hide credentials.
- Command injection can provide direct access to the underlying operating system.
- SUID binaries are important targets during Linux privilege escalation enumeration.
- Tools such as `strings` can be misused when they run with elevated privileges.
